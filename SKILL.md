# 预约会议室

通过本仓库脚本调用飞书会议室与日历能力完成**初始化配置**与**按忙闲自动预约**；底层依赖已配置好的 `lark-cli` 与对应 API 权限。

## 配置与入口

**仓库根目录**（含 `conf/`、`scripts/`）下执行 Python 脚本。

| 能力 | 入口命令 | 说明 |
|------|----------|------|
| 初始化层级与会议室列表 | `python scripts/meeting_init_processor.py` | **须由用户在本机终端自行执行**（交互式 `input()`，AI 不得代跑）。在仓库根目录运行，按提示选城市 → 职场 → 楼层，写入 `conf/meeting.json` |
| 发起预约 | `python scripts/meeting_processor.py --start-time <ISO> --end-time <ISO> --summary "<主题>"` | 可由 AI 在用户确认参数后代为执行。按配置中会议室顺序查忙闲，首个空闲则创建日程并挂载会议室 |

可选参数（预约脚本）：`--description`（日程描述）、`--config <路径>`（默认 `conf/meeting.json`）、`--calendar-id`（默认 `primary`）。

可选参数（初始化脚本）：`--top-n`（城市候选数量，默认 10）、`--page-size`（层级/会议室分页大小，默认 100）。

**首次使用前：**确保本机 `lark-cli` 已登录且具备会议室/日历相关权限；若用户未配置，先按 `lark-shared` 技能完成 `lark-cli config init` 与 `lark-cli auth login`。

**初始化与预约的分工：**初始化依赖终端交互输入，**AI 只负责告知命令、工作目录、前置条件与流程说明**；用户完成后可告知 AI「已初始化」，再继续预约。预约脚本无交互，AI 可在用户确认时段与主题后执行。

## Capabilities
1. 初始化会议室配置
这是使用会议室预约功能的前置步骤，需要在首次使用或更换办公地点时执行。

**执行方式（强制）：**由**用户**在本地打开终端，进入本技能所在仓库根目录，执行 `python scripts/meeting_init_processor.py`，在终端内按编号或名称回答脚本提问。**禁止**由 AI 代为执行该初始化命令（无法完成交互、且易与用户的 `lark-cli` 身份/环境不一致）。


2. 智能预约会议室
这是最核心的场景：从自然语言中解析时段与主题，转换为脚本所需的 ISO 时间后执行预约。

**使用场景：**
- "明天下午3点订个会议室开项目周会"
- "下周一上午10点，讨论需求，1小时"
- "30分钟后订个会议室"

### 必填参数（调用脚本前必须齐全）

| 参数 | CLI 对应 | 要求 |
|------|-----------|------|
| 开始时间 | `--start-time` | ISO 8601，**精确到秒**，勿含毫秒/微秒；示例 `2026-04-05T15:00:00+08:00` |
| 结束时间 | `--end-time` | 同上；须**晚于**开始时间 |
| 会议主题 | `--summary` | 非空字符串 |

**业务约束（脚本会校验）：**单次预约时长须 **≤ 4 小时**；时间须为整秒（`.000` 可省略，但不要带小数秒）。

### AI 解析时间与引导话术

用户常用自然语言描述时间，**由 AI 结合当前日期/时区（默认按中国东八区 `+08:00`）解析**，再拼出传给脚本的 ISO 字符串。

1. **从用户话里提取：**开始锚点（如「明天下午 3 点」「下周一 10:00」「30 分钟后」）、**时长**或**结束锚点**（如「1 小时」「到 5 点」）。若只说了开始未说时长/结束，**必须追问**：「会议大概开多久？」或「结束时间是几点？」
2. **换算为 ISO：**开始、结束均转为 `YYYY-MM-DDTHH:MM:SS+08:00`（或与用户明确约定的时区）；相对时间（如「30 分钟后」）以「当前时刻」为基准计算。
3. **回显确认：**在执行脚本前用一句话向用户确认解析结果，例如：「将预约 **2026-04-06 15:00–16:00（+08:00）**，主题：**项目周会**，是否确认？」避免歧义日/夜、跨日未说明等情况。
4. **缺主题时：**直接问「会议主题写什么？」不能为空。

**核心原则：**
- 预约前检查 `conf/meeting.json` 是否存在；不存在或需重配时，**口头/书面引导用户自行在终端运行** `python scripts/meeting_init_processor.py`，并说明须在仓库根目录、需已登录 `lark-cli`；**不要**由 AI 执行初始化脚本。
- 参数不全或时间有歧义时，**逐项补问**，不要猜测用户未说的时长或结束时间。
- 预约按配置中 **`rooms` 列表顺序**查忙闲，在首个空闲会议室上创建日程并挂载该会议室（非仅按 `selected_level` 模糊搜索）。


## Example Usage

**初始化（用户侧说法 → 你给出命令与步骤，用户自己去终端执行）：**
- 用户：「帮我初始化会议室」→ 回复：进入仓库根目录后执行 `python scripts/meeting_init_processor.py`，按终端里城市/职场/楼层列表用编号或名称选择；完成后可说一声再继续预约。**不要**替用户跑该命令。
- 用户：「我在北京，怎么配会议室？」→ 同上，可补充：若列表里前 10 个城市没有北京，输入名称后脚本会尝试全量匹配。

**预约（你解析时间后替用户拼好命令或等价调用）：**
- 用户：「明天下午 3 点订个会议室开项目周会」
  → 解析出起止 ISO + 主题后执行：
  `python scripts/meeting_processor.py --start-time "2026-04-06T15:00:00+08:00" --end-time "2026-04-06T16:00:00+08:00" --summary "项目周会"`
  （若用户未说时长，应先问清再定 `end-time`。）

## Scripts

- `scripts/meeting_init_processor.py`：交互式初始化，写入 `conf/meeting.json`（层级 + `rooms`）。**仅用户本机终端执行，AI 不调用。**
- `scripts/meeting_processor.py`：读配置、查忙闲、创建日程并预约会议室；**必填** `--start-time`、`--end-time`、`--summary`。
- `scripts/lark_cli.py`：飞书 API 封装（由上述脚本引用，一般不单独对用户暴露）。

## Limitations
- 依赖 `lark-cli` 可用且具备对应 API 权限
- 会议室层级命名可能重名，必须通过二次确认消歧
